## 🎯 The Problem SiteForge AI Solves

* **Rigid & Black-Box Builders:** Most AI builders generate code you cannot customize easily. SiteForge AI features a built-in **Monaco Editor Console** that allows you to inspect, modify, download, or publish the raw HTML/CSS/JS output.
* **Complex Server Setup for Lead Capture:** Static landing pages generated by AI traditionally need backend work to capture user forms. SiteForge AI dynamically injects custom API endpoints into the generated template to store leads in a managed database.
* **Loss of State on Page Refresh:** Custom interactive components (counters, carts, toggle items) lose state on refresh. SiteForge AI automatically structures state management using browser `localStorage` on output layouts.
* **Delayed Deployments:** Offers instant publishing—generating a public URL for your layout in one click without configuring separate static hosting providers.

## ⚙️ Getting Started & Local Setup

### 1. Prerequisites
* **Node.js** (v18 or higher)
* **MongoDB** (Local instance or MongoDB Atlas Connection String)
* **Stripe** and **Firebase** accounts (Optional for basic local development due to Developer Bypass)

### 2. Clone and Install Dependencies

```bash
# Clone the repository
git clone https://github.com/shivamsharma75/SiteForge-AI.git
cd SiteForge-AI

# Install frontend dependencies
cd client
npm install

# Install backend dependencies
cd ../server
npm install
```

### 3. Environment Setup

#### Server Configuration
Create a `.env` file in the **`server`** directory:
```env
PORT=8000
MONGODB_URL=your_mongodb_connection_string
JWT_SECRET=your_jwt_signature_secret
OPENROUTER_API_KEY=your_openrouter_api_key
FRONTEND_URL=http://localhost:5173
STRIPE_SECRET_KEY=your_stripe_secret_key
STRIPE_WEBHOOK_SECRET=your_stripe_webhook_signing_secret
```

#### Client Configuration
Create a `.env` file in the **`client`** directory:
```env
VITE_FIREBASE_API_KEY=your_firebase_web_api_key
```

> [!TIP]
> **Firebase & Google Sign-In Bypass:** If you do not have Firebase configured, you can run the app on `localhost` and click the **"Dev Bypass Login (Localhost only)"** button in the sign-in modal to immediately access all editor features.

---

### 4. Running the Application

You need to run both the frontend developer server and the backend API concurrently:

```bash
# Run Backend (Runs on http://localhost:8000)
cd server
npm run dev

# Run Frontend (Runs on http://localhost:5173)
cd client
npm run dev
```

Open your browser and navigate to **`http://localhost:5173`** to start building.

## 💳 Testing Stripe Webhooks Locally

To verify Stripe payment credit updates locally:

1. Download the [Stripe CLI](https://stripe.com/docs/stripe-cli).
2. Authenticate the CLI:
   ```bash
   stripe login
   ```
3. Forward webhook events to your local server:
   ```bash
   stripe listen --forward-to localhost:8000/api/stripe/webhook
   ```
4. Copy the webhook signing secret returned by the CLI (starts with `whsec_`) and set it as `STRIPE_WEBHOOK_SECRET` in your server `.env` file.
5. Restart your backend server. Trigger a checkout in test mode, and credits will update automatically!
